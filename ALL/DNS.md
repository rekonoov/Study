### üí° What is DNS

DNS (Domain Name System) is a system that translates human-readable domain names, like tryhackme.com, into IP addresses, like 104.26.10.229, so devices can find each other on the Internet.

**Analogy:**  
Like a home address is needed for delivering mail, an IP address is needed for routing data to the correct computer. Instead of remembering numbers, we use names such as tryhackme.com.

---

### üß© Domain Name Structure

#### 1. TLD (Top-Level Domain)

Located at the end of the domain, e.g., tryhackme.com ‚Üí TLD = .com.  

Types of TLDs:  

- gTLD ‚Äî generic domains: .com, .org, .edu, .gov  
- ccTLD ‚Äî country domains: .ru, .jp, .de, .co.uk, .ca  
  There are over 2000 TLDs, including new ones like .club, .biz, .online.

#### 2. Second-Level Domain

The main site name before the TLD, e.g., tryhackme.com ‚Üí tryhackme.  

Rules:  

- max 63 characters  
- allowed: a-z, 0-9, -  
- cannot start/end with -, no consecutive hyphens

#### 3. Subdomain

Located to the left of the second-level domain, e.g., admin.tryhackme.com ‚Üí subdomain = admin. Multiple subdomains allowed, e.g., jupiter.servers.tryhackme.com.  

Rules:  

- ‚â§ 63 characters  
- only a-z, 0-9, -  
- cannot start/end with -  
- total domain length ‚â§ 253 characters

---

### üß± DNS Record Types

- **A** ‚Äî points to IPv4 address, e.g., 104.26.10.229  
- **AAAA** ‚Äî points to IPv6 address, e.g., 2606:4700:20::681a:be5  
- **CNAME** ‚Äî alias to another domain, e.g., store.tryhackme.com ‚Üí shops.shopify.com  
- **MX** ‚Äî mail server, e.g., alt1.aspmx.l.google.com (with priority)  
- **TXT** ‚Äî text data such as SPF, DKIM, verification, e.g., v=spf1 include:_spf.google.com ~all  

**Note:**  
MX and TXT records are often used for spam protection, domain ownership verification, and mail routing.

---

### ‚öôÔ∏è How DNS Query Works

1. **Local cache check** ‚Äî computer checks its cache. If the record exists, it's used without accessing the Internet.  
2. **Recursive DNS server** ‚Äî query is sent to ISP server, which may have cached records.  
3. **Root DNS server** ‚Äî if not found, the recursive server queries the root DNS server, which knows where TLD servers are.  
4. **TLD server** ‚Äî forwards the query to the authoritative name server. Example for tryhackme.com: kip.ns.cloudflare.com, uma.ns.cloudflare.com.  
5. **Authoritative DNS server** ‚Äî stores the real DNS records and returns the IP to the recursive server.  
6. **Caching** ‚Äî recursive server stores the record for TTL (Time To Live) in seconds, reducing network load.  
7. **Response to user** ‚Äî browser connects directly to the IP address.
